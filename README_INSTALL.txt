===============================================
   환자 주소 분석기 설치 가이드
===============================================

📋 목차
1. 설치 전 준비사항
2. 설치 방법 (Windows/Mac)
3. 사용 방법
4. 문제 해결

===============================================

1️⃣ 설치 전 준비사항
---------------

✅ Python 3.9 이상 설치 필요
   - Windows: https://www.python.org/downloads/
   - Mac: 기본 설치되어 있음 (확인: 터미널에서 python3 --version)

✅ 인터넷 연결 필요 (최초 1회)
   - 라이브러리 설치 및 주소 분석 시 필요

===============================================

2️⃣ 설치 방법
---------------

📁 이 폴더 통째로 복사/이동 가능합니다
   - 바탕화면, 문서, 어디든 OK

===============================================

3️⃣ 사용 방법
---------------

【 Mac 사용자 】
1. start.sh 파일을 더블클릭
2. "확인되지 않은 개발자" 경고 시:
   - 우클릭 → "열기" 선택
   - 또는 터미널에서: ./start.sh
3. 웹 브라우저 자동 실행

【 Windows 사용자 】
1. start.bat 파일을 더블클릭
2. Windows Defender 경고 시: "추가 정보" → "실행"
3. 웹 브라우저 자동 실행

【 웹 화면에서 】
1. "파일 선택" 버튼 클릭
2. 환자 주소 엑셀/CSV 파일 업로드
   ⚠️ 반드시 '주소' 컬럼 필요!
3. "🚀 분석 시작" 버튼 클릭
4. 결과 확인 후 "📥 결과 다운로드" 클릭

===============================================

4️⃣ 파일 형식 요구사항
---------------

✅ 지원 형식: .csv, .xlsx, .xls

✅ 필수 컬럼: "주소" (정확히 이 이름)

✅ 예시:
   주소
   경기도 화성시 동탄구 동탄신리천로8길 15
   경기도 화성시 동탄구 동탄순환대로19길 59

❌ 불가:
   환자명, 진료 내역 등 다른 정보 있어도 OK
   단, '주소' 컬럼은 필수!

===============================================

5️⃣ 문제 해결
---------------

【 앱이 실행되지 않아요 】
→ Python 설치 확인: 터미널/명령 프롬프트에서
   python3 --version
   (3.9 이상이어야 함)

【 "주소 컬럼이 없습니다" 오류 】
→ 엑셀 첫 줄에 "주소" 컬럼명 확인
   (띄어쓰기, 오타 주의!)

【 분석이 너무 느려요 】
→ 정상입니다. 100건 기준 약 30초 소요
   (API 호출 제한 때문)

【 일부 주소가 분석 안 돼요 】
→ Postcodify API가 인식 못 하는 주소
   (신축 건물, 오타 등)
   → 수동 확인 필요
