# 요구사항 정의

## 배경

- **고객**: 동탄 병원 원장 (친구)
- **문제**: 환자 주소가 도로명 주소로만 저장되어 있어 지역별 마케팅 분석이 어려움
- **목표**: 지도 기반 환자 분포 시각화 (동/아파트 단위)

## 상세 요구사항

### 1. 주소 변환
- 도로명 주소 → 동 (예: 신동, 목동)
- 도로명 주소 → 아파트명 (예: 000아파트)
- 도로명 주소 → 위경도 좌표

### 2. 시각화
- 지도 위에 환자 수 집계
- 동별/아파트별 통계
- 정적 HTML 리포트 (월 1회 생성 가정)

### 3. 데이터 흐름
```
[병원 EMR] → 엑셀 추출 → geocode.py → 지오코딩 CSV → visualize.py → HTML 지도
```

## 기술 스택 (초안)

| 역할 | 기술 |
|------|------|
| 지오코딩 API | Kakao Local API (무료 300,000건/일) |
| 데이터 처리 | Python (pandas) |
| 지도 시각화 | Folium (Python) |
| 의존성 관리 | pip / requirements.txt |

## 제약사항

- **개인정보**: 환자 이름/진료 내역은 제외, 주소만 처리
- **데이터 보안**: 원본 데이터는 Git 외부 (data/raw/.gitignore)
- **비용**: 무료 API 범위 내 (Kakao: 30만건/일)

## 성공 조건

- [ ] 주소 1,000건 → 지오코딩 성공률 95% 이상
- [ ] 지도 HTML 파일 10초 이내 생성
- [ ] 동별/아파트별 집계 정확도 확인 (샘플 10건 수동 검증)

## 향후 고려사항 (Phase 2)

- 병원 EMR 시스템에 "동", "아파트명" 필드 추가 자문
- 실시간 대시보드 (Streamlit / Tableau)
- 재방문율, 진료과목 등 추가 지표
