# 요구사항 정의

## 배경

- **고객**: 동탄 병원 원장 (친구)
- **문제**: 환자 주소가 도로명 주소로만 저장되어 있어 지역별 마케팅 분석이 어려움
- **목표**: 지도 기반 환자 분포 시각화 (동/아파트 단위)

## 상세 요구사항

### 1. 주소 변환
- 도로명 주소 → 동 (예: 신동, 목동)
- 도로명 주소 → 아파트명 (예: 000아파트)
- 도로명 주소 → 위경도 좌표

### 2. 시각화
- HTML 정적 리포트 (동별/아파트별 순위)
- 요약 통계 (총 환자 수, 고유 동 수, 아파트 거주자 수)
- 반응형 테이블 (상위 20개)

### 3. 데이터 흐름
```
[병원 EMR] → CSV/엑셀 추출 → geocode.py (Postcodify) → CSV → visualize.py → HTML 리포트
```

## 기술 스택 (확정)

| 역할 | 기술 |
|------|------|
| 지오코딩 API | Postcodify (무료, API 키 불필요) |
| 데이터 처리 | Python (pandas) |
| 리포트 시각화 | HTML/CSS (자체 템플릿) |
| 의존성 관리 | pip / requirements.txt |

## 제약사항

- **개인정보**: 환자 이름/진료 내역은 제외, 주소만 처리
- **데이터 보안**: 원본 데이터는 Git 외부 (data/raw/.gitignore)
- **비용**: 무료 API 사용 (Postcodify: 일일 제한 있지만 충분)

## 성공 조건

- [x] 주소 7건 샘플 → 지오코딩 성공률 100%
- [ ] HTML 리포트 10초 이내 생성
- [ ] 동별/아파트별 집계 정확도 확인
